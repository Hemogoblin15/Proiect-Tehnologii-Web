Manners Matter

Autori

Ghidoarca Mihail & Pletosu Razvan

Abstract

MaMa (Manners Matter) este o aplicatie web de invatare a bunelor maniere pe internet care ofera utilizatorilor diverse lectii in functie de ocupatia lor. Utilizatorii au optiunea de a-si configura profilul cu datele lor. Cu ajutorul acestora, platforma le va sugera lectii relevante profilului creat.

Introducere

Scopul acestei aplicatii web este de a oferi utilizatorilor, printr-o interfata user-friendly, o oportunitate de dezvoltare personala. In timpul crearii unui cont, utilizatorul isi poate configura profilul, adaugandu-si ocupatia si tara. Cu ajutorul optiunilor alese, platforma ii va sugera clientului cele mai potrivite lectii.
Prin intermediul platformei, utilizatorul poate sa citeasca lectii si sa dea teste in urma acestora.
Dupa ce un test a fost dat scorul va fi memorat si utilizatorul va aparea intr-un clasament global.

Structura

Pentru a realiza interfata web si pentru a implementa functionalitatile aplicatiei folosim:
3.1 HTML

HyperText Markup Language (HTML) este un limbaj de marcare utilizat pentru crearea paginilor web ce pot fi afisate intr-un browser (sau navigator). Scopul HTML este mai degraba prezentarea informatiilor – paragrafe, fonturi, tabele s.a.m.d. – decat descrierea semanticii documentului.

3.2 CSS

CSS (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atasa elementelor HTML prin intermediul unor fisiere externe sau direct in cadrul documentului html.

3.3 Javascript

JavaScript (JS) este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor. Este folosit mai ales pentru introducerea unor functionalitati în paginile web, codul JavaScript din aceste pagini fiind rulat de catre browser.
Proiectul contine modulul "scripts", in care utilizam Fetch API's.

3.4 NodeJS

Node.js este un mediu de execuție JavaScript de fundal multiplatformă cu sursă deschisă, care rulează pe motorul V8 și execută cod JavaScript în afara unui navigator web.
Pentru a putea fi rulat, trebuie instalate dependentele ("npm install") si pentru a porni serverul, se foloseste comanda "npm start".

3.5 MongoDB

MongoDB este o bază de date NoSQL open-source orientată pe documente. Acestă bază de date beneficiază de suport din partea companiei 10gen. MongoDB face parte din familia de sistemelor de baze de date NoSQL. Baza de date MaMa contine colectia "users".

3.6 Github

GitHub este un serviciu de gazduire web pentru proiecte de dezvoltare a software-ului care utilizeaza sistemul de control al versiunilor Git.

Interfata Aplicatiei

4.1 Index

Utilizatorul trebuie sa fie autentificat pentru a avea acces la functionalitatile site-ului. Ulterior autentificarii, utilizatorul va avea acces la lectii personalizate, va avea optiunile de a se deconecta (Logout) – fiind redirectionat catre pagina de login sau de a trimite mailuri de ajutor (Help).
Pagina beneficiaza de un design responsive, navbar-ul fiind strans intr-un "hamburger"– vizibil doar prin apasarea butonului.

4.2 Lessons

Aceasta pagina este destul de asemanatoare din punct de vedere al design-ului, fiind de asemenea responsive cu ajutorul proprietatilor grid-ului.

4.3 Register

Pe pagina de register, utilizatorul va trebui sa completeze cele 6 campuri (Firstname, LastName, Email, Country, Ocupation, Password) pentru a-si crea contul. Campurile trebuie neaparat completate, altfel va aparea o eroare care reaminteste utilizatorului ca acele campuri nu trebuie omise. Odata ce toate campurile sunt completate, utilizatorul va apasa pe butonul de Register unde va fi redirectionat pe pagina de Login.

4.4 Login

Pe pagina de login, utilizatorul are posibilitatea de a se loga pe contul sau care a fost creat pe pagina de Register. Cele doua campuri (Email, Password), asemanator paginii de Register, trebuie sa fie completate, altfel clientul va fi atentionat ca acele campuri nu trebuie lasate goale si butonul nu va functiona. Daca utilizatorul nu are un cont, atunci va apasa pe butonul de go to Register pentru a-si crea unul, altfel va apasa pe butonul de Login iar, daca informatiile din form sunt corecte cu cele salvate din baza de date, atunci el va fi redirectionat pe pagina de Home.

4.5 About

Pe pagina about este o scurta descriere a site-ului.

4.6 Help

Pe pagina help, utilizatorul va completa un formular asemanator cu cele de login si register, cu campurile Email, Subject si Message, astfel apasand butonul de Send un mail va fi trimis catre emailul de ajutor (daca toate campurile au fost completate conform).
